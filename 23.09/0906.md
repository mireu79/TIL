#### 학습활동
- **Image views** - 투명하거나 불투명한 배경에 이미지를 표시한다.
- **Tab bar** - 표시줄 항목을 사용해 동일한 보기에서 콘텐츠 창사이를 탐색한다.
- **Label** - 읽고 복사할수 있지만 편집할수 없는 텍스트이다.
- **Button** - 각 제어 가능한 기능에 접근할 수 있게 한다.
- **Toggle** - 각 상태를 나타내는 다른 모양을 사용해 꺼짐, 켜짐과 같은 반대상태중 선택한다.
- **Status bar** -  화면 상단 가장자리에 나타내며, 시간, 배터리잔량 등 현재상태를 표시한다.
- **Navigation bar** - 창이나 화면 상단에 나타나 콘텐츠 계층구조 탐색에 도움준다.
- **Modality** - 상위 뷰와의 상호 작용을 방지하고 해제하려면 명시적인 작업이 필요한 별도의 집중 모드로 콘텐츠를 표시하는 디자인 기술입니다. 하나의  페이지에서 하나의 작은 다른 페이지를 띄워 표현했습니다.
- **Virtual Keyboards** - 데이터를 입력하는 데 사용할 수 있는 가상키보드를 제공한다.

사람들은 Apple의 접근성 기능을 사용하여 자신에게 맞는 방식으로 기기와 상호 작용하는 방식을 개인화합니다 
- Best pratice
 1. 접근성을 염두에 두고 디자인하세요(단순성 과 인지성을 우선시) 
2. 개인화를 지원합니다. 환경적 변화에 적응하도록 환경을 디자인 
3. 앱이나 게임의 접근성을 심사하고 테스트하세요.
4. 일반적인 상호작용에는 단순화된 동작을 선호합니다.
- 제스처
1. 제스처 기반 작업을 수행하는 대체 방법을 제공합니다.
2. 가능하다면 두 가지 이상의 물리적 상호작용을 통해 앱의 핵심 기능에 액세스할 수 있도록 하세요.(캡쳐할때)
3. 사용자 정의 제스처를 정의하는 경우 사람들이 앱과 상호 작용할 수 있는 대체 방법을 제공하는 보조 기술을 지원해야 합니다.
4. iOS 또는 iPadOS 앱에서 드래그 앤 드롭이 가능하도록 만드세요.
- 버튼, 컨트롤
1. 모든 컨트롤과 대화형 요소에 충분히 큰 히트 타겟을 지정하세요.
2. 맞춤 요소의 접근성을 특성화합니다.
3. 일관된 스타일 계층 구조를 사용하여 버튼의 상대적 중요성을 전달합니다.
4. 시스템에서 제공하는 스위치 구성 요소를 선호합니다.
5. 색상 외에 밑줄과 같은 시각적 표시기를 링크에 제공하는 것을 고려해보세요.
- 사용자입력
1. 사람들이 타이핑이나 몸짓 대신 말로 정보를 입력할 수 있게 하세요.
2. 음성만으로 중요한 작업을 수행할 수 있도록 Siri 또는 바로가기를 지원합니다.
3. 가능하다면 사람들이 일반 텍스트를 선택하는 것을 막지 마십시오.
- 햅틱
1. 가능한 경우 시스템 정의 햅틱을 지원합니다.  예를 들어, 시스템 앱은 작업이 성공하거나 실패했을 때 또는 이벤트가 곧 발생할 때 사람들에게 알리기 위해 햅틱을 재생합니다. 사람들이 혼동하지 않도록 시스템 정의 햅틱을 앱에서 일관되게 사용하세요.
- 텍스트 표시
1. 현재 글꼴 크기에 관계없이 일관된 정보 계층 구조를 유지합니다.
2. 앱에서 보통 또는 무거운 글꼴 두께를 선호하세요
3. 사람들이 굵은 텍스트를 켰을 때 앱이 올바르게 반응하고 보기 좋게 보이는지 확인하세요
4. 사용자 정의 글꼴을 읽을 수 있는지 확인하십시오.
5. 사용자 정의 글꼴을 읽을 수 있는지 확인하십시오.
6. 긴 텍스트 구절에는 이탤릭체나 모두 대문자를 사용하지 마세요. 
- 색상 및 효과
1. 개체를 구별하거나 중요한 정보를 전달하기 위해 색상에만 의존하지 마십시오.
2. 텍스트에는 시스템 색상을 선호합니다.
3. 두 가지 상태나 값을 구별하는 유일한 방법으로 색상 조합을 사용하지 마십시오. 
4. 뷰가 색상 반전에 올바르게 응답하는지 확인하세요.
5. 가독성을 높이려면 강하게 대비되는 색상을 사용하세요.

읽기 쉬운 텍스트를 보장하는 것 외에도 활자체 선택은 정보 계층 구조를 명확하게 하고, 중요한 콘텐츠를 전달하고, 브랜드를 표현하는 데 도움이 될 수 있습니다.
- Best pratice
1. 대부분의 사람들이 쉽게 읽을 수 있는 최소 글꼴 크기를 유지하도록 노력하십시오.
2. 중요한 정보를 강조하고 사람들이 계층 구조를 시각화하는 데 도움이 되도록 필요에 따라 글꼴 두께, 크기 및 색상을 조정합니다.
3. 인터페이스에서 사용하는 서체의 수를 최소화하세요. 
4. 다양한 상황에서 가독성을 테스트합니다.
5. 일반적으로 가독성을 유지하려면 가벼운 글꼴 두께를 피하세요.
6. 텍스트 크기 변경에 대응할 때 중요한 콘텐츠의 우선순위를 지정하세요.

2. 필요한 경우 내장된 텍스트 스타일을 수정합니다.예를 들어, 굵은 특성은 텍스트에 가중치를 추가하여 또 다른 수준의 계층 구조를 만들 수 있게 해줍니다. 가독성을 높이거나 공간을 절약해야 하는 경우 기호 특성을 사용하여 행간을 조정할 수도 있습니다. 예를 들어, 넓은 열이나 긴 구절에 텍스트를 표시할 때 줄 사이에 더 많은 공간( 느슨한 행간 )을 사용하면 사람들이 한 줄에서 다음 줄로 이동하는 동안 자신의 위치를 ​​더 쉽게 유지할 수 있습니다. 반대로 높이가 제한된 영역(예: 목록 행)에 여러 줄의 텍스트를 표시해야 하는 경우 줄 사이의 간격을 줄입니다 () 텍스트가 잘 맞는 데 도움이 될 수 있습니다. 3줄 이상의 텍스트를 표시해야 하는 경우 높이가 제한되는 영역에서도 촘촘한 행간을 피하세요.
3. 인터페이스 모형에서 필요에 따라 추적을 조정합니다.
